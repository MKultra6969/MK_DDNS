# MK_DDNS

[![Python 3.12+](https://img.shields.io/badge/python-3.10+-blue.svg)](https://www.python.org/downloads/release/python-3100/)
[![Docker](https://img.shields.io/badge/docker-%230db7ed.svg?style=for-the-badge&logo=docker&logoColor=white)](https://www.docker.com/)
[![License: WTFPL](https://img.shields.io/badge/license-WTFPL-red.svg)](./LICENSE.md)

Telegram-бот на Python (`aiogram` + `aiohttp`) для автоматического обновления DDNS (Dynamic DNS) записей у [1984Hosting](https://1984.hosting/) и Cloudflare. Бот отслеживает изменение вашего внешнего IP-адреса и автоматически обновляет A-записи для всех добавленных доменов.

Все управление происходит через Telegram, а основной сценарий работы вынесен в кнопки и меню. Slash-команды сохранены как запасной и сервисный вариант, а для развертывания используется Docker Compose.

## 🚀 Возможности

-   **Автоматическое отслеживание IP**: Бот периодически проверяет ваш внешний IP и обновляет DNS-записи только при его изменении.
-   **Два провайдера в одном интерфейсе**: Один бот управляет `1984Hosting FreeDNS` и `Cloudflare`.
-   **Управление через Telegram**: Настройка провайдеров, добавление и удаление записей, проверка статуса и запуск обновления выполняются прямо из чата.
-   **Поддержка Cloudflare**: Бот получает список зон через API и позволяет сохранить запись в режиме `proxy` или `DNS only`.
-   **Принудительное обновление и статус**: Можно вручную запустить синхронизацию и посмотреть последний известный IP, состояние ключей и количество записей.
-   **Миграция старой базы**: При первом запуске новая версия импортирует старые данные из `OldDB/ddns.db` или `ddns.db`, если они существуют.
-   **Безопасность**: Бот реагирует только на команды от администратора, указанного в `ADMIN_ID`.
-   **Надежность**: Для хранения настроек используется SQLite, а для запуска и обновления удобно использовать Docker Compose.

## ⚙️ Установка и запуск

### Шаг 1: Подготовка

1.  **Клонируйте репозиторий**:
    ```bash
    git clone https://github.com/<your-username>/MK_DDNS.git
    cd MK_DDNS
    ```

2.  **Получите API key от 1984Hosting**:
    -   Зайдите в панель управления [1984Hosting](https://1984.hosting/).
    -   Перейдите в раздел **FreeDNS**.
    -   Сгенерируйте ваш **API Key**.

3.  **Подготовьте Cloudflare API token**, если используете Cloudflare:
    -   Создайте токен с правами **Zone:Read** и **DNS:Edit**.
    -   Сохраните его, позже бот проверит токен перед записью в настройки.

4.  **Создайте Telegram-бота**:
    -   Найдите в Telegram бота `@BotFather`.
    -   Создайте нового бота с помощью команды `/newbot`.
    -   Сохраните полученный **TELEGRAM_TOKEN**.

5.  **Узнайте свой Telegram User ID**:
    -   Найдите в Telegram бота `@userinfobot`.
    -   Напишите ему `/start`, и он пришлет ваш `Id`.

### Шаг 2: Конфигурация

1.  Скопируйте `.env.example` в `.env`:
    ```bash
    cp .env.example .env
    ```

2.  Откройте файл `.env` и вставьте ваши данные:
    ```env
    TELEGRAM_TOKEN=123456:ABC-DEF1234ghIkl-zyx57W2v1u123456789
    ADMIN_ID=123456789
    CHECK_INTERVAL=300
    ```
    -   `TELEGRAM_TOKEN`: Токен вашего бота.
    -   `ADMIN_ID`: Ваш ID из Telegram.
    -   `CHECK_INTERVAL`: Интервал проверки IP в секундах (по умолчанию 300 = 5 минут).

3.  Ключ `1984Hosting` и токен `Cloudflare` задаются уже после запуска бота через раздел `Провайдеры`.

### Шаг 3: Запуск

Убедитесь, что у вас установлен Docker и Docker Compose. Выполните команду:

```bash
docker compose up -d --build
```

Бот запустится в фоновом режиме. Чтобы посмотреть логи, используйте команду:
`docker logs -f mk_ddns_bot`

## 💬 Использование

Найдите вашего бота в Telegram и начните с ним диалог.

1.  `/start` — Запуск и открытие главного меню.

2.  `Провайдеры` или `/providers` — Раздел для сохранения секретов провайдеров:
    -   `1984Hosting API key`, если используете `1984Hosting`.
    -   `Cloudflare API token`, если используете `Cloudflare`.

3.  `Записи` или `/records` — Раздел управления DNS-записями:
    -   для `1984Hosting` бот запросит полный домен;
    -   для `Cloudflare` бот предложит выбрать зону, имя записи и режим `proxy` или `DNS only`.

4.  `/list` — Показывает список всех записей и кнопки удаления с подтверждением.

5.  `Обновить сейчас` или `/check` — Запускает принудительную проверку IP и обновление всех записей. Полезно для первой синхронизации и диагностики.

6.  `Статус` или `/status` — Показывает последний известный IP, состояние ключей провайдеров, количество записей и интервал проверки.

7.  `Главное меню` или `/menu` — Возвращает пользователя к основному меню.

8.  `Отмена ввода` или `/cancel` — Сбрасывает текущий ввод и возвращает клавиатуру.
